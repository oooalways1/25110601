## 🔍 문제 진단 방법

### 1단계: 브라우저 개발자 도구 열기
- `F12` 키 누르기
- 또는 마우스 우클릭 → "검사" 또는 "Inspect"

### 2단계: Console 탭 확인
빨간색 에러 메시지가 있는지 확인:

#### 정상적인 경우:
- 에러 없음
- 또는 경고(노란색)만 있음

#### 문제가 있는 경우:
- 빨간색 에러 메시지
- "Failed to load resource" 같은 메시지
- "404 Not Found" 메시지

### 3단계: Network 탭 확인
1. Network 탭 클릭
2. 페이지 새로고침 (`F5`)
3. 빨간색으로 표시된 파일이 있는지 확인

## 🛠️ 해결 방법

### 방법 1: 브라우저 캐시 완전 삭제

**Chrome/Edge**:
1. `Ctrl + Shift + Delete`
2. "전체 기간" 선택
3. "캐시된 이미지 및 파일" 체크
4. "쿠키 및 기타 사이트 데이터" 체크
5. "데이터 삭제" 클릭
6. 브라우저 완전히 종료 후 재시작

### 방법 2: 시크릿/프라이빗 모드

**Chrome/Edge**:
```
Ctrl + Shift + N
```

**Firefox**:
```
Ctrl + Shift + P
```

시크릿 창에서 게임 URL 접속

### 방법 3: 강력 새로고침

페이지에서:
- **Windows**: `Ctrl + Shift + R` 또는 `Ctrl + F5`
- **Mac**: `Cmd + Shift + R`

### 방법 4: 다른 기기/네트워크에서 테스트

- 스마트폰으로 접속
- 다른 컴퓨터로 접속
- 모바일 데이터로 접속 (WiFi 끄기)

## 🔄 여전히 안 된다면?

### 재배포 방법

```powershell
# 1. dist 폴더 삭제
if (Test-Path "dist") { Remove-Item -Recurse -Force "dist" }

# 2. 다시 빌드
npm run build

# 3. .nojekyll 파일 추가
New-Item -Path "dist\.nojekyll" -ItemType File -Force

# 4. 배포
npx gh-pages -d dist -t
```

### GitHub Pages 재설정

1. https://github.com/oooalways1/25110601/settings/pages 접속
2. Source를 **None**으로 변경
3. **Save** 클릭
4. 다시 Source를 **gh-pages**로 변경
5. **Save** 클릭
6. 5분 정도 기다리기
