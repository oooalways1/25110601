# PRD: 수학 산성비 게임 (Math Rain Game)

## 1. 프로젝트 개요

### 1.1 목적
초등학교 3학년 학생들이 곱셈과 나눗셈을 재미있게 학습할 수 있는 교육용 게임을 개발한다.

### 1.2 타겟 사용자
- **주 사용자**: 초등학교 3학년 학생 (만 8-9세)
- **부 사용자**: 학부모, 교사

### 1.3 핵심 가치
- 게임을 통한 자연스러운 수학 학습
- 반복 학습을 통한 연산 능력 향상
- 성취감을 통한 학습 동기 부여

## 2. 기능 요구사항

### 2.1 핵심 기능

#### 2.1.1 게임 모드 선택
- **곱셈 모드**: 2~9단 곱셈 문제
- **나눗셈 모드**: 기본 나눗셈 문제 (나머지 없는 나눗셈)
- **혼합 모드**: 곱셈과 나눗셈 랜덤 출제

#### 2.1.2 난이도 설정
- **쉬움**: 
  - 곱셈: 2, 3, 5단
  - 나눗셈: 2, 5로 나누기
  - 낙하 속도: 느림
  - 문제 출현 빈도: 낮음
  
- **보통**: 
  - 곱셈: 2~5단
  - 나눗셈: 2~5로 나누기
  - 낙하 속도: 중간
  - 문제 출현 빈도: 중간
  
- **어려움**: 
  - 곱셈: 2~9단
  - 나눗셈: 2~9로 나누기
  - 낙하 속도: 빠름
  - 문제 출현 빈도: 높음

#### 2.1.3 게임 플레이
- **문제 낙하**: 화면 상단에서 수학 문제가 비처럼 떨어짐
- **답 입력**: 
  - 키보드로 답 입력
  - 엔터키로 제출
  - 백스페이스로 수정
- **정답 처리**: 
  - 정답 시: 해당 문제 소멸 + 점수 획득 + 효과음
  - 오답 시: 입력창 흔들림 + 재입력 기회
- **게임 오버**: 문제가 화면 하단(지면)에 닿으면 생명 감소

#### 2.1.4 점수 시스템
- **기본 점수**: 정답 1개당 10점
- **콤보 시스템**: 연속 정답 시 보너스 점수
  - 3연속: 1.5배
  - 5연속: 2배
  - 10연속: 3배
- **시간 보너스**: 빠른 답변 시 추가 점수
- **생명**: 3개 시작, 모두 소진 시 게임 종료

### 2.2 부가 기능

#### 2.2.1 학습 리포트
- 게임 종료 후 결과 화면 표시
  - 총 점수
  - 정답률
  - 최고 콤보
  - 틀린 문제 목록
  - 취약한 구구단/나눗셈 영역 분석

#### 2.2.2 기록 관리
- 최고 점수 저장
- 플레이 히스토리
- 학습 진행도 그래프

#### 2.2.3 보상 시스템
- 레벨업 시스템
- 업적/배지 획득
- 캐릭터 커스터마이징 요소 언락

### 3.2 디자인 가이드라인

#### 3.2.1 색상
- **주 색상**: 밝고 친근한 파스텔 톤
- **배경**: 하늘색 그라데이션
- **문제**: 구름 모양의 흰색 박스
- **정답**: 초록색 효과
- **오답**: 빨간색 효과

#### 3.2.2 폰트
- **문제 텍스트**: 크고 명확한 고딕체 (최소 24px)
- **숫자**: 가독성 높은 폰트
- **UI 텍스트**: 둥근 고딕체

#### 3.2.3 애니메이션
- 문제 낙하: 부드러운 하강 애니메이션
- 정답 시: 반짝임 + 소멸 효과
- 오답 시: 좌우 흔들림
- 콤보 달성: 화려한 이펙트

### 3.3 사운드

#### 3.3.1 효과음
- 정답: 긍정적인 "띠링" 소리
- 오답: 부드러운 "땡" 소리
- 콤보: 점점 높아지는 음계
- 게임 오버: 하강하는 음계
- 생명 감소: 경고음

#### 3.3.2 배경음악
- 밝고 경쾌한 BGM
- 음소거 옵션 제공

## 4. 기술 요구사항

### 4.1 플랫폼
- **우선순위 1**: 웹 브라우저 (HTML5)
  - Chrome, Edge, Safari 최신 버전 지원
  - 반응형 디자인 (태블릿 지원)
  
- **우선순위 2**: 데스크톱 애플리케이션 (선택사항)

### 4.2 기술 스택 (권장)
- **프론트엔드**: 
  - HTML5 Canvas / WebGL
  - JavaScript (또는 TypeScript)
  - React 또는 Vue.js (선택사항)
  
- **게임 엔진**: 
  - Phaser.js
  - PixiJS
  - 또는 바닐라 JavaScript
  
- **데이터 저장**: 
  - LocalStorage (클라이언트 측)
  - IndexedDB (대용량 데이터)

### 4.3 성능 요구사항
- 초기 로딩 시간: 3초 이내
- 프레임률: 60 FPS 유지
- 입력 지연: 50ms 이내
- 메모리 사용: 100MB 이하

## 12. 부록

### 12.1 문제 출제 범위

#### 곱셈
```
2단: 2×1 ~ 2×9
3단: 3×1 ~ 3×9
4단: 4×1 ~ 4×9
5단: 5×1 ~ 5×9
6단: 6×1 ~ 6×9
7단: 7×1 ~ 7×9
8단: 8×1 ~ 8×9
9단: 9×1 ~ 9×9
```

#### 나눗셈
```
2로 나누기: 2÷2, 4÷2, 6÷2, ..., 18÷2
3으로 나누기: 3÷3, 6÷3, 9÷3, ..., 27÷3
4로 나누기: 4÷4, 8÷4, 12÷4, ..., 36÷4
5로 나누기: 5÷5, 10÷5, 15÷5, ..., 45÷5
6으로 나누기: 6÷6, 12÷6, 18÷6, ..., 54÷6
7로 나누기: 7÷7, 14÷7, 21÷7, ..., 63÷7
8로 나누기: 8÷8, 16÷8, 24÷8, ..., 72÷8
9로 나누기: 9÷9, 18÷9, 27÷9, ..., 81÷9
```

### 12.2 콤보 보너스 상세
| 연속 정답 | 점수 배율 | 화면 효과 |
|-----------|----------|-----------|
| 1-2개 | 1.0배 | 기본 |
| 3-4개 | 1.5배 | "Good!" 표시 |
| 5-7개 | 2.0배 | "Great!" 표시 + 별 효과 |
| 8-9개 | 2.5배 | "Excellent!" 표시 + 불꽃 효과 |
| 10개 이상 | 3.0배 | "Amazing!" 표시 + 무지개 효과 |
